Doc Formatting:

 - GET /api/examples/docs?number={INPUT}&type={INPUT}
    
   - :: The start of a request URL.
   GET :: Type of request. Possible values: GET, POST, PATCH, DELETE, PUT, HEAD, OPTIONS
   /api/examples/docs :: Path. Always starts with /api/. Will be specified for each Endpoint
   ? :: Represents the start of a query. In requests with it listed it is always required.
   number= :: The query
   {INPUT} :: The user given input for the query
   & :: Represents the start of another query. 
   type= :: The second query

URL Shortener APIs:

 - GET /api/urls/submit?longurl={INPUT}

   Returns a response containing the submited longurl, resulting shorturl, and url-id
  
   {INPUT} :: A HTTP or HTTPS link. 
   
   
 - GET /api/urls/listed
    
   Returns an array of all urls currently in the DB. Each entry contains the longurl, shorturl, and url-id

Quote API

 - GET /api/quote

  Returns a response containing a random quote
